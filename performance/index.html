性能优化

性能优化主要从以下几个方面入手：

1. 构建性能

2. 传输性能

3. 运行性能

1. 构建性能

构建性能是指:从打包开始到打包完毕或者从修改之后点击保存自动打包到页面的更新过程,整个过程都是在开发阶段进行的

开发阶段优化了构建性能，可以大大的提高打包速度，从而提升了开发效率

构建性能主要可以从以下几个方面考虑：

1. 减少模块解析

webpack中模块解析包括：抽象语法树分析、依赖分析、模块语法替换

不进行解析的模块会怎样？

不做解析的模块，在loader处理之后就直接输出

不做解析的模块，就少了上面的步骤，从而提升了打包速度

模块不需要解析的前提条件：

模块中无其他依赖，如：jquery

配置如下：

        module.exports = {
            //...
            module: {
                noParse: /jquery|lodash/,  // 不解析jqery和loadsh
            }
        };

2. 传输性能

传输性能是指：文件从服务端传输到客户端的过程所需要的时间

传输性能的提升可以缩短从页面请求到呈现的时间，可以从以下几个方面考虑：

2.1 总传输量：将所有的传输文件内容都加起来就是总传输

2.2 文件数量：需要传输的所有文件数量

2.3 浏览器缓存： 通过设置可以使得浏览器缓存传输的文件，被缓存的文件再次获取就会从缓存中获取
